# Traffic Model

Проект реализует генерацию и анализ сетевого трафика для двух моделей: равномерной и пуассоновской. Трафик представляется как последовательность дискретных пакетов, у каждого пакета есть время прихода от начала симуляции и размер.

C++-часть отвечает за генерацию трафика и сохранение результата в CSV. Python-часть читает CSV-файл, рассчитывает характеристики трафика и строит гистограммы задержек и размеров пакетов.

## Назначение файлов

### `cpp/main.cpp`

Основной исходный файл C++-части проекта.

Содержит:

- чтение входного файла;
- проверку базовых параметров симуляции;
- класс `EqualTrafficModel` для равномерной модели трафика;
- класс `PoissonTrafficModel` для пуассоновской модели трафика;
- генерацию CSV-файла с результатами симуляции.

### `cpp/CMakeLists.txt`

Файл конфигурации CMake для сборки C++-приложения компилятором GCC.

### `cpp/traffic_model`

Собранный бинарный файл C++-приложения. Используется для запуска проекта без дополнительной сборки.

### `python/analyzer.py`

Python-скрипт для анализа CSV-файла, созданного C++-приложением.

Скрипт рассчитывает:

- название модели трафика;
- средний размер пакета;
- средний битрейт;
- среднюю задержку между пакетами;
- гистограмму задержек между пакетами;
- гистограмму размеров пакетов.

### `start_traffic_model.sh`

Bash-скрипт для последовательного запуска C++- и Python-частей проекта одной командой.

### `examples/`

Папка с примерами входных файлов для равномерной и пуассоновской моделей.

### `result/`

Папка с результатами работы проекта. Файлы в этой папке перезаписываются при новом запуске.

## Модели трафика

### Равномерная модель

В равномерной модели все пакеты имеют одинаковый размер, а задержка между соседними пакетами фиксирована.

Формат второй строки входного файла:

```text
equal: <package_size> <delay>
```

Параметры:

- `package_size` — размер каждого пакета в байтах (натуральное число);
- `delay` — фиксированная задержка между пакетами в секундах (неотрицательное число с плавающей точкой).

Пример:

```text
174.89
equal: 6 13.04
```

### Пуассоновская модель

В модели `poisson` моделируется пуассоновский поток событий. Количество пакетов на фиксированном интервале времени имеет распределение Пуассона, поэтому интервалы между соседними пакетами моделируются экспоненциальным распределением. Размер пакета также задаётся экспоненциальным распределением.

Формат второй строки входного файла:

```text
poisson: <avg_package_size> <avg_delay>
```

Параметры:

- `avg_package_size` — средний размер пакета в байтах (неотрицательное число с плавающей точкой);
- `avg_delay` — средняя задержка между пакетами в секундах (неотрицательное число с плавающей точкой).

Пример:

```text
234.87
poisson: 5 8.5
```

## Формат входного файла

Входной файл состоит из двух строк.

Первая строка — время симуляции в секундах. Значение должно быть неотрицательным числом с плавающей точкой и не должно превышать `3600`.

Вторая строка — название модели и параметры модели:

```text
<model_name>: <param_1> <param_2>
```

Поддерживаются модели:

- `equal`;
- `poisson`.

## Формат выходного CSV

C++-приложение сохраняет результат в файл:

```text
result/result_csv.csv
```

CSV-файл содержит два столбца:

```text
time,size
```

где:

- `time` — время прихода пакета от начала симуляции;
- `size` — размер пакета.

Разделитель в CSV-файле — запятая.

## Результаты анализа

После запуска проекта в папке `result/` создаются или обновляются файлы:

```text
result/result_csv.csv

result/histograms.png
```

### `result/result_csv.csv`

CSV-файл со сгенерированным трафиком.

### `result/analysis.json`

JSON-файл с рассчитанными характеристиками:

- `model` — название модели трафика;
- `avg_package_size` — средний размер пакета;
- `avg_bitrate` — средний битрейт;
- `avg_delay` — средняя задержка между пакетами.

### `result/histograms.png`

Изображение с двумя гистограммами:

- гистограмма задержек между пакетами;
- гистограмма размеров пакетов.

## Требования

Проект рассчитан на запуск под Linux.

Для запуска без сборки нужны:

- Python 3;
- библиотеки Python:
  - `pandas`;
  - `matplotlib`.
- CMake;
- GCC с поддержкой C++17.

## Установка Python-зависимостей

Если зависимости не установлены, выполните:

```bash
python3 -m pip install pandas matplotlib
```

## Запуск без сборки

В архив уже включён собранный бинарный файл C++-приложения:

```text
cpp/traffic_model
```

Поэтому для проверки проекта сборка не требуется.

Сначала выдайте права на запуск bash-скрипта:

```bash
chmod +x start_traffic_model.sh
```

Запуск равномерной модели:

```bash
./start_traffic_model.sh examples/input_equal.txt
```

Запуск пуассоновской модели:

```bash
./start_traffic_model.sh examples/input_poisson.txt
```

После запуска результаты будут сохранены в папку `result/`.

## Пример полного запуска

```bash
python3 -m pip install pandas matplotlib
chmod +x start_traffic_model.sh

./start_traffic_model.sh examples/input_equal.txt
cat result/analysis.json

./start_traffic_model.sh examples/input_poisson.txt
cat result/analysis.json
```

## Примечание

Файлы в папке `result/` перезаписываются при каждом новом запуске проекта.

При запуске скрипта на других данных, необходимо их внести в уже имеющиеся файлы в папке `examples/`, либо сохранить свой файле в этой же папке.
